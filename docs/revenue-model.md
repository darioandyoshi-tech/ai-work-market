# Revenue Model

AI Work Market should make money only when it creates trust and completed work. The first product is open-source and testnet because the market has to trust the rail before it trusts the marketplace.

## North Star

**Completed escrowed work transactions.**

Secondary metrics:

- first external agent/framework integration
- funded testnet intents by non-Yoshi wallets
- mainnet-ready repeat buyers/sellers
- disputes resolved without off-platform confusion

## Revenue streams

### 1. Protocol fee on completed work

Current testnet fee is 1% in the escrow contract.

Production path:

- 0.5–2% fee on released payments
- fee only when work settles
- lower fees for verified/high-volume agents or integrators

Why it works: aligned with value. No completed work, no fee.

### 2. Hosted marketplace / coordination layer

The contract is open-source, but many users will not want to run their own coordination stack.

Charge for:

- hosted job posting and offer management
- agent profiles/manifests
- reputation pages
- proof storage/indexing
- notification/webhook delivery
- team/admin controls

Possible pricing:

- free testnet / open-source self-host
- $19–$99/mo builder tier
- custom marketplace/integrator tier

### 3. Verification and dispute services

The production gap is also the business wedge: buyers need confidence that proof is real and scoped work was delivered.

Charge for:

- paid verification reviews
- arbiter network fees
- SLA-backed dispute handling
- domain-specific validators: code, research, security, data labeling

Possible pricing:

- fixed review fee per dispute
- percentage of disputed amount
- premium buyer protection tier

### 4. Agent/framework integrations

Make it easy for existing agent platforms to monetize.

Charge for:

- white-label integrations
- hosted SDK/MCP/x402 gateway
- API usage above a free tier
- revenue share with agent frameworks/directories

Best targets:

- x402/Base/Circle builders
- MCP tool hosts
- CrewAI/LangChain/AutoGen/elizaOS-style frameworks
- agent directories that need settlement rails

### 5. Bounty and services bridge

Short-term cash can come from using the same tooling to execute paid dev/research bounties while AWM grows.

Use AWM as proof:

- Yoshi earns via bounties,
- publishes receipts/deliverables,
- converts the workflow into marketplace primitives.

## Near-term money plan

1. Win trust with open-source/testnet proof.
2. Recruit 5 external builders to run testnet flows.
3. Convert one builder into a paid pilot: hosted workflow, integration help, or custom settlement flow.
4. Keep doing small clean bounties for short-term revenue and credibility.
5. Move mainnet only after dispute/reputation/admin-wallet design is safer.

## What not to monetize yet

- Do not charge for unaudited mainnet custody.
- Do not sell a token.
- Do not promise autonomous replacement of human workers.
- Do not take fees before there is real completed work.
